# フロントエンド ライブラリ比較

**最終更新:** 2026/03/14  
**対象:** カンバンボード（DnD）、ガントチャート

---

## 1. ドラッグ&ドロップ（カンバンボード用）

カンバンボード（S-07-02: PJ進捗管理ボード）でIssueカードをドラッグしてステータス変更する機能に必要。

### 比較表

| | **@dnd-kit** | **Pragmatic DnD** (Atlassian) | **react-beautiful-dnd** |
|---|---|---|---|
| **GitHub Stars** | 14k+ | 9k+ | 34k |
| **週間DL数** | ~8.5M | ~1.5M | ~3M |
| **メンテナンス** | 活発 (v6.3.1, 2024/12) | 活発 (Atlassian管理) | **非推奨 (deprecated)** |
| **バンドルサイズ** | ~26KB (min+gzip, core+sortable) | ~20KB (min+gzip) | ~30KB (min+gzip) |
| **React 18対応** | ✅ | ✅ | ⚠️ 公式サポート終了 |
| **Next.js対応** | ✅ Pages/App Router両対応 | ✅ | ⚠️ SSR問題あり |
| **TypeScript** | ✅ ネイティブ | ✅ ネイティブ | ✅ |
| **キーボード操作** | ✅ 組み込み | ✅ 組み込み | ✅ |
| **タッチ対応** | ✅ | ✅ | ✅ |
| **カスタマイズ性** | 高い（低レベルAPI） | 高い（低レベルAPI） | 中（opinionated） |
| **学習コスト** | 中 | 中 | 低 |
| **カンバン実績** | 多数（公式例あり） | Trello/Jira/Confluence で使用 | 多数 |
| **ライセンス** | MIT | Apache 2.0 | Apache 2.0 |

### 各ライブラリの特徴

#### @dnd-kit
- **長所:**
  - 最もダウンロード数が多く、コミュニティが大きい
  - `@dnd-kit/core` + `@dnd-kit/sortable` の構成でカンバンに最適
  - Next.js Pages Router での実装例が豊富
  - `DndContext` → `SortableContext` → `useSortable` のシンプルなAPI
  - カラム間のアイテム移動が標準サポート
- **短所:**
  - 低レベルAPIのため、react-beautiful-dndよりボイラープレートが多い
  - DragOverlay の実装が必要

#### Pragmatic DnD (Atlassian)
- **長所:**
  - Atlassianが Trello / Jira / Confluence で実際に使用
  - フレームワーク非依存（React以外でも使える）
  - パフォーマンスに強い最適化
- **短所:**
  - dnd-kitと比べるとコミュニティが小さい
  - React向けのドキュメント・例がdnd-kitより少ない
  - Atlassian特有の設計思想への理解が必要

#### react-beautiful-dnd ❌
- **長所:**
  - 最も簡単に実装できる
  - 直感的なAPI
- **短所:**
  - **2024年に正式にdeprecated**（新規プロジェクトでは非推奨）
  - React 18 Strict Mode で問題あり
  - SSRでの問題が未修正のまま
  - セキュリティパッチも出ない可能性

### 推奨: **@dnd-kit**

| 理由 |
|------|
| 最もアクティブにメンテナンスされている |
| Next.js Pages Routerでの実績豊富 |
| Tailwind CSS との組み合わせ例が多い |
| カンバンボードの公式サンプルあり |
| TypeScript ネイティブ対応 |
| `@dnd-kit/sortable` でカラム間DnDが簡潔に書ける |

**必要パッケージ:**
```
@dnd-kit/core
@dnd-kit/sortable
@dnd-kit/utilities
```

## 2. ガントチャート

プロジェクト進捗一覧（S-05-02）でタスクの予実をタイムライン上に可視化する機能に必要。

> **Note:** ガントチャートのUI/デザインは、別途Claudeブラウザ版で生成した参考資料（`docs/ui-references/gantt-chart/`）を基に判断する。ライブラリの採否はデザイン確定後に最終決定する。

### 比較表

| | **frappe-gantt** | **DHTMLX Gantt** | **SVAR React Gantt** | **自前実装** |
|---|---|---|---|---|
| **GitHub Stars** | 5.8k | 1.7k | 新しい | — |
| **メンテナンス** | 活発 (v1.2.2, 2026/03) | 活発 (v9.1.2, 2026/03) | 活発 | — |
| **ライセンス** | MIT | GPL v2 (無料) / 商用有料 | MIT (Core) / PRO有料 | — |
| **バンドルサイズ** | ~15KB (min+gzip) | ~200KB+ | 不明 | 最小 |
| **React対応** | ラッパー経由 (`frappe-gantt-react`) | ラッパー経由 | ネイティブReact | — |
| **Next.js対応** | ✅ (`frappe-gantt-next`) | ⚠️ SSR注意 | ✅ | ✅ |
| **タスク依存関係表示** | ✅ | ✅ | ✅ (PRO) | 自前 |
| **ドラッグでスケジュール変更** | ✅ | ✅ | ✅ | 自前 |
| **進捗率表示** | ✅ | ✅ | ✅ | 自前 |
| **日/週/月ビュー切替** | ✅ | ✅ | ✅ | 自前 |
| **リソース管理** | ❌ | ✅ | ✅ (PRO) | 自前 |
| **エクスポート (PDF/PNG)** | ❌ | ✅ (PRO) | ❌ | 自前 |
| **カスタマイズ性** | 中 (CSS) | 高 (API豊富) | 高 | 完全 |
| **学習コスト** | 低 | 中〜高 | 中 | 高 |
| **日本語対応** | ✅ (ロケール設定) | ✅ | ✅ | — |

### 各ライブラリの特徴

#### frappe-gantt
- **長所:**
  - 軽量（~15KB）で依存関係なし
  - MITライセンスで完全無料
  - SVGベースで見た目がきれい
  - シンプルなAPIで学習コストが低い
  - Next.js用パッケージ (`frappe-gantt-next`) あり
- **短所:**
  - 機能が基本的（エクスポートやリソース管理なし）
  - Reactラッパーが薄い（公式ではない）
  - 大量タスク（500+）でのパフォーマンス不明

#### DHTMLX Gantt
- **長所:**
  - 最も機能が豊富（オートスケジューリング、リソース管理、エクスポート）
  - エンタープライズ実績豊富
  - 10k+タスクのパフォーマンス実証済み
- **短所:**
  - **GPL v2ライセンス**（商用利用は有料ライセンスが必要）
  - バンドルサイズが大きい（200KB+）
  - ReactラッパーがネイティブReactではない
  - 価格: $599/開発者〜 (Commercial)

#### SVAR React Gantt
- **長所:**
  - ネイティブReactコンポーネント（ラッパーではない）
  - MIT Core + PRO版の構成
  - MCP Server対応（AI開発支援）
- **短所:**
  - 比較的新しく、コミュニティが小さい
  - PRO機能（依存関係、クリティカルパス）は有料
  - 日本語の情報が少ない

#### 自前実装（CSS Grid / SVG）
- **長所:**
  - デザインの完全な制御
  - 不要な機能を含まない（バンドルサイズ最小）
  - Claudeの参考資料を忠実に再現可能
- **短所:**
  - 開発工数が大きい（DnD、ズーム、リサイズ等すべて自前）
  - バグの温床になりやすい
  - テスト工数も大きい

### 推奨: チームで要件確定後に決定

ガントチャートはUI参考資料の確認後に最終判断すべきだが、暫定的な推奨は以下。

| シナリオ | 推奨 | 理由 |
|---------|------|------|
| 基本的なガントでOK（MVP） | **frappe-gantt** | 軽量、無料、十分な機能 |
| カスタムデザイン重視 | **自前実装** | 参考資料を忠実に再現 |
| 将来的に高機能化したい | **SVAR React Gantt** | ネイティブReact + 拡張性 |
| エンタープライズ要件 | **DHTMLX Gantt** | 最も機能豊富（要予算） |

## 3. その他検討が必要なライブラリ

### 3.1 日付ピッカー

Issue作成・PJ作成で期限設定に必要。

| | **react-datepicker** | **react-day-picker** | **@mui/x-date-pickers** |
|---|---|---|---|
| **週間DL数** | ~5M | ~2M | ~2M |
| **バンドルサイズ** | ~30KB | ~15KB | 大 (MUI依存) |
| **カスタマイズ** | 中 (CSS) | 高 (ヘッドレス) | 高 |
| **日付範囲選択** | ✅ | ✅ | ✅ |
| **Tailwind相性** | 中 | ✅ 良い | 低 |

**推奨: react-day-picker** — 軽量、ヘッドレスでTailwindとの相性が良い。

### 3.2 チャート（将来）

ワークロード可視化や分析で必要になる可能性。

| | **recharts** | **Chart.js (react-chartjs-2)** | **visx** |
|---|---|---|---|
| **週間DL数** | ~5M | ~3M | ~500K |
| **バンドルサイズ** | ~50KB | ~60KB | 小 (モジュラー) |
| **学習コスト** | 低 | 低 | 高 |
| **カスタマイズ** | 中 | 中 | 高 |

**推奨: recharts** — React向けに設計、学習コスト低、十分な機能。

### 3.3 クラス名ユーティリティ

条件付きTailwindクラスの結合に必要。

| | **clsx** | **classnames** | **tailwind-merge** |
|---|---|---|---|
| **バンドルサイズ** | ~0.3KB | ~0.4KB | ~3KB |
| **機能** | クラス結合 | クラス結合 | クラス結合+Tailwind競合解決 |

**推奨: clsx + tailwind-merge** — `clsx` で条件結合、`tailwind-merge` でTailwindクラスの競合を解決。
